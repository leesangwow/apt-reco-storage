# Handoff: 비슷한집 — 관심 아파트 가격대 유사 단지 추천

## Overview
"비슷한집"은 사용자가 **관심 아파트를 선택/검색**하면, 그 단지와 **비슷한 매매가대**의 다른 단지를 주변/타지역에서 3~4개씩 추천하는 모바일 서비스다.
추천 범위는 **우리 동네 / 우리 구 / 주변 시 / 서울 전체 / 지역 직접 선택(전국)** 으로 사용자가 고를 수 있고, 결과는 **가격차·거리·평형·준공년도** 기준으로 정렬(오름/내림 토글)된다. 한 번에 3곳씩 보여주고 `더보기`로 누적 표시한다.

목표: **웹 먼저 출시 → 동일 코드/데이터로 앱 출시.**

## About the Design Files
이 번들의 파일(`Apartment Recommender.dc.html`)은 **HTML로 만든 디자인 레퍼런스(프로토타입)** 이다. 의도한 화면 모양·인터랙션을 보여주기 위한 것이지, 그대로 가져다 쓰는 프로덕션 코드가 아니다.
작업의 목표는 이 HTML 디자인을 **타겟 코드베이스의 환경(아래 권장 스택)으로 재구현**하는 것이다. `.dc.html`은 자체 런타임(`support.js`) 위에서 도는 단일 파일 컴포넌트 형식이므로, 로컬에서는 **브라우저로 열어 시각/동작 확인용**으로만 쓰고, 실제 구현은 Next.js(React) 컴포넌트로 옮긴다.

> `.dc.html`을 로컬에서 열어보려면: 이 폴더를 통째로 두고 `Apartment Recommender.dc.html`을 브라우저로 열면 된다(같은 폴더의 `support.js`, 인터넷의 Pretendard 폰트를 로드).

## Fidelity
**High-fidelity (hifi).** 최종 색상·타이포·간격·인터랙션이 확정된 목업이다. UI는 픽셀 단위로 재현하되, 스타일은 타겟 코드베이스의 방식(예: Tailwind/디자인 시스템)으로 정리해 구현한다.

이 프로토타입은 **3가지 카드 스타일 시안(A/B/C)을 나란히 비교**하도록 만들어졌다. 출시 시에는 **그 중 1개를 선택**해 단일 화면으로 구현한다(나머지는 참고용).

## Screens / Views

기본 화면은 하나(추천 결과 화면)이며, **카드 영역만 다른 3가지 변형(A/B/C)** 으로 제시된다. 공통 구조는 동일하다.

### 공통 레이아웃 (모바일, 폭 390px 기준)
위에서 아래로:
1. **상태바** (시간 9:41 + 신호/배터리 아이콘) — 데모용, 실제 앱에선 OS 상태바
2. **내 관심 아파트 헤더** (카드) — "내 관심 아파트" 뱃지 + `검색·변경` 버튼, 단지명 / 위치·평형·준공년도 / 매매가
3. **범위 칩 (가로 스크롤)** — `우리 동네` `우리 구` `주변 시` `서울 전체` + `＋ 지역`(전국 추가)
4. **서브타이틀** — "<범위 라벨> · 비슷한 가격대 N곳"
5. **정렬 칩 (가로 스크롤)** — `정렬` 라벨 + `가격차순` `거리순` `평형순` `준공순` (활성 칩에 ↑/↓)
6. **추천 카드 리스트** — 3개씩 (변형 A/B/C에서 카드 모양만 다름)
7. **`더보기` 버튼** — 3곳씩 누적, 전부 보면 `접기`
8. (오버레이) **검색·변경 바텀시트**, **지역 추가 바텀시트**

### Variant A — 리스트형 (당근 피드 느낌)
- 카드: 가로 배치. 좌측 92×92 썸네일(라운드 13px), 우측 정보.
- 우측: 단지명(15px/800) + 우상단 하트(♡/♥) / 위치·거리(12px 회색) / **매매가(17px/800) + 가격차 뱃지** / 메타(11px 회색).
- 가격차 뱃지: `0.3억 저렴해요`(초록 #16A06A, 배경 #E2F5EC) / `0.4억 높아요`(주황 #E8552D, 배경 #FBEAE2) / `내 관심과 동일`(회색).

### Variant B — 카드형 (토스 카드 느낌)
- 카드: 세로. 상단 풀폭 썸네일(높이 128px) 위에 — 우상단 원형 하트 버튼, 좌하단 **가격차 텍스트(반투명 검정 pill, 흰 글자)**.
- 하단 패딩 영역: 단지명(16px/800) + 매매가(19px/800) 한 줄 / 위치·거리 / 상단 구분선 + 메타.

### Variant C — 비교형 (가격차 한눈에)
- 카드: 상단에 작은 44×44 썸네일 + 단지명·하트 + 위치·거리, 우측 매매가(16px/800).
- 그 아래 **가격차 막대(track 140px)**: 가운데 기준선(69px), 내 관심보다 싸면 **왼쪽 초록 막대**, 비싸면 **오른쪽 주황 막대**(폭 = |가격차|×52px, 최대 66px). 막대 옆 가격차 텍스트.
- 하단 구분선 + 메타(평형·준공·세대수).
- 상단에 범례: `◂ 더 저렴 … 더 비쌈 ▸`.

### 바텀시트 — 검색·변경
- 헤더 핸들 + "관심 아파트 선택" 타이틀.
- 검색 입력(돋보기) — 단지명 부분일치 필터.
- 결과 리스트(관심목록 겸용): 각 행 = 단지명 / "구 동 · 평형" / 매매가, 탭하면 기준 아파트 변경 → 모든 가격차 재계산.

### 바텀시트 — 지역 추가 (＋ 지역)
- 헤더: "지역 추가 (N곳 선택됨)" + 노란 `완료` 버튼.
- **시/도 탭(가로 스크롤)**: 서울·경기·인천·부산·대구·대전.
- **시군구 목록**: 각 행 = 시군구명 / "M개 단지" / 우측 `＋`(미선택) 또는 `✓`(선택, 노란 테두리·연노랑 배경).
- 탭 토글로 추가/해제. 추가하면 상단 범위 칩에 해당 시군구 칩이 생기고 즉시 선택됨. 해제 시 현재 보던 지역이면 `서울 전체`로 폴백.

## Interactions & Behavior
- **범위 칩 클릭** → `scope` 변경, 추천 풀 교체, `더보기` 카운트 3으로 리셋.
- **＋ 지역** → 지역 추가 시트 오픈. 시군구 토글로 `addedRegions` 추가/제거, 추가 시 그 지역 자동 선택.
- **정렬 칩 클릭**:
  - 다른 정렬 선택 → 그 정렬의 기본 방향 적용(아래 표).
  - 같은 정렬 재클릭 → 방향 토글(asc ↔ desc). 활성 칩에 ↑(asc)/↓(desc) 표시.
- **검색·변경** → 검색 시트. 입력은 단지명 부분일치, 선택 시 `baseId` 변경 → 가격차 전체 재계산, 시트 닫힘.
- **하트** → 카드별 관심 토글(로컬 상태).
- **더보기** → `visible += 3`. 다 보이면 라벨 `접기`, 누르면 다시 3으로.
- 바텀시트: 배경 오버레이(rgba(20,20,18,.34)) 탭 또는 `완료`로 닫힘. 등장 애니메이션 `translateY(100%)→0`, 0.26s `cubic-bezier(.22,1,.36,1)`.
- 칩/카드 영역은 가로 스크롤(스크롤바 숨김). 모바일 단일 컬럼.

### 정렬 기준 & 기본 방향
| 정렬 | 비교값 | 기본 방향 |
|---|---|---|
| 가격차순(diff) | `abs(price - myPrice)` | asc(차이 작은 순) |
| 거리순(dist) | `distance_km` | asc(가까운 순) |
| 평형순(area) | `pyeong` | desc(큰 평형 순) |
| 준공순(year) | `year` | desc(신축 순) |
- 기준 아파트(`baseId`)와 이름이 같은 단지는 추천에서 제외.

## State Management
필요한 상태(현 프로토타입 기준):
- `baseId` — 기준(관심) 아파트 id
- `scope` — `'dong' | 'gu' | 'city' | 'seoul' | '<시도>/<시군구>'`
- `addedRegions` — 추가한 전국 지역 `[{id, sido, gu}]`
- `sort` — `'diff'|'dist'|'area'|'year'`, `sortDir` — `'asc'|'desc'`
- `visible` — 현재 표시 개수(3의 배수)
- `liked` — 관심 토글 맵 `{ [aptId]: boolean }`
- `searchOpen` / `q` — 검색 시트 열림 / 검색어
- `regionOpen` / `sheetSido` — 지역 시트 열림 / 선택된 시/도

데이터 페칭(실서비스): 기준 아파트 + scope + sort + page를 파라미터로 추천 API 호출(아래 "추천 로직" 참고). 현 프로토타입은 모두 클라이언트 목 데이터.

## 추천 로직 (목업 → 실제 쿼리)
```sql
select * from apt
where abs(price - :myPrice) <= :band        -- 비슷한 가격대
  and region in (:scopeRegions)             -- 범위 칩
  and id <> :myId                           -- 기준 단지 제외
order by
  case :sort
    when 'diff' then abs(price - :myPrice)
    when 'dist' then distance_km
    when 'area' then pyeong
    when 'year' then year
  end  /* :dir = asc|desc */
limit 3 offset :page * 3;                    -- 더보기
```
